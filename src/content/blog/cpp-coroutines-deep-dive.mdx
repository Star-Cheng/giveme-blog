# C++ 20 Coroutines 深度解析

协程（Coroutines）是 C++20 引入的一个重要特性，它提供了一种轻量级的协作式多任务机制。本文将深入探讨协程的工作原理、实现细节以及最佳实践。

## 协程的基本概念

协程是一种可以被暂停和恢复执行的函数。与普通函数不同，协程可以在执行过程中挂起（suspend），然后在后续时间点恢复（resume）执行。

```cpp
#include <coroutine>
#include <iostream>

struct HelloCoroutine {
    struct promise_type {
        HelloCoroutine get_return_object() {
            return HelloCoroutine{ std::coroutine_handle<promise_type>::from_promise(*this) };
        }
        std::suspend_always initial_suspend() { return {}; }
        std::suspend_always final_suspend() noexcept { return {}; }
        void return_void() {}
    };

    std::coroutine_handle<promise_type> handle;
};

HelloCoroutine say_hello() {
    std::cout << "Hello ";
    co_await std::suspend_always{};
    std::cout << "World!" << std::endl;
}

int main() {
    auto task = say_hello();
    std::cout << "Coroutine created" << std::endl;
    task.handle.resume();
    std::cout << "Coroutine resumed" << std::endl;
    task.handle.destroy();
    return 0;
}
```

## Promise Type

每个协程必须关联一个 promise type，它负责管理协程的生命周期：

- `get_return_object()`: 返回协程的调用者将得到的对象
- `initial_suspend()`: 协程开始前是否自动挂起
- `final_suspend()`: 协程结束后是否自动挂起
- `return_value()` / `return_void()`: 处理协程的返回值

## 协程帧的内存布局

协程帧（Coroutine Frame）是协程状态的存储区域，包含：

1. **Promise 对象** - 存储协程的返回值和状态
2. **局部变量** - 协程挂起时需要保存的变量
3. **挂起点** - 恢复执行时从哪里继续
4. **参数副本** - 函数参数的副本

## 最佳实践

### 1. 使用 RAII 管理协程句柄

```cpp
class scoped_coroutine {
    std::coroutine_handle<> handle_;
public:
    explicit scoped_coroutine(std::coroutine_handle<> h) : handle_(h) {}
    ~scoped_coroutine() { if (handle_) handle_.destroy(); }
    scoped_coroutine(const scoped_coroutine&) = delete;
    scoped_coroutine& operator=(const scoped_coroutine&) = delete;
};
```

### 2. 避免在析构函数中挂起

析构函数中执行 `co_await` 是未定义行为。

### 3. 合理使用 `noexcept`

标记为 `noexcept` 的协程在抛出异常时会调用 `std::terminate`。

## 总结

C++20 协程为异步编程提供了一种高效的解决方案，但也带来了复杂性。理解协程的底层机制对于写出正确且高效的代码至关重要。

> 协程的编译转换是复杂的，建议使用成熟的库（如 ASIO、cppcoro）来简化开发。
